# 《光环生存》项目记忆

## 项目定位

- 游戏名：《光环生存》（暂定），英文名：Aura Survivor。
- 类型：2D Roguelike / 割草生存 / 单机休闲 / 自动攻击。
- 引擎：Cocos Creator 3.8 LTS。
- 语言：TypeScript。
- 首发平台：Windows。
- 后续平台：Android。
- 开发规模：1 人，预计 2 到 3 个月业余开发。

- `docs/project-proposal.md`：游戏立项书。

## 核心玩法

玩家控制角色移动，角色周围持续存在光环。怪物不断刷新并追击玩家，进入光环范围后持续受到伤害。怪物死亡掉落经验球，玩家拾取经验后升级，并从三个随机强化中选择一个继续成长。

核心循环：

```text
移动 -> 刷怪 -> 追击 -> 光环伤害 -> 怪物死亡 -> 经验掉落 -> 拾取升级 -> 技能三选一 -> 更强怪物 -> 继续生存
```

游戏目标：撑过 8 分钟即普通胜利；420 秒出现裂隙核心，提前击破可获得完美胜利。

## MVP 范围

V1.0 最小可玩版本只保留这些内容：

- 玩家 WASD / 方向键移动。
- 一个持续伤害光环。
- 普通怪自动追击。
- 怪物死亡和经验掉落。
- 经验升级和三选一强化。
- 生存计时与游戏结束。
- 一张无限循环地图，并按当前关卡名切换主题地表、障碍和色调。

暂不优先做：

- 多角色。
- 装备系统。
- 天赋树。
- 成就系统。
- 独立多地图加载和复杂地图事件。
- 排行榜。

## 产品方向记录

产品定位不做“低配吸血鬼幸存者”，而是做“走位驱动的光环构筑游戏”。核心卖点不是自动攻击本身，而是玩家通过移动和站位，把怪物拖进自己的危险光环里，同时避免被包围。

一句话定位：

```text
《光环生存》是一款无武器、靠站位和光环构筑割草的短局 Roguelite 生存游戏。
```

当前最重要的验证目标：

- 开局 30 秒内让玩家理解玩法。
- 3 分钟内让玩家获得明显成长反馈。
- 5 分钟后让局势开始紧张。
- 8 分钟形成一次完整高潮和胜利结算。

当前 vertical slice 决策：

- 局长目标：撑过 8 分钟即胜利。
- 差异化机制：光环效果优先作为局内升级，不先绑定不同角色。
- 当前光环流派：脉冲负责周期爆发与跨边界充能，寒霜负责 72% 内圈减速，压缩是一次性贴身高伤路线。
- 脉冲与寒霜可共存并触发碎冰；压缩与脉冲/寒霜互斥，选择后移除普通范围牌。
- 外沿甜点区获得 `1.5×` 普通光环伤害，玩家需要在外沿输出、寒霜内圈控制和压缩贴身之间做站位选择。
- 强化拥有等级和上限，首次升级必含行为牌，每局提供一次重投；经验增幅牌已移出 8 分钟卡池。
- 压力模型：基础怪、快怪、厚怪、自爆怪、裂隙眼和风剪灵均已落地，阶段事件使用五种波次模板和精英奖励组织节奏。
- 公平性基线：全局受击保护、特殊怪冷却/在场上限、连续刷怪曲线、经验球合并上限、真实局时钟与受控模拟步长分离均已落地。
- 结算区分普通胜利、Boss 完美胜利和失败，并记录构筑、Boss 状态、死因、伤害、等级与击杀。
- 本地纪录保存最长生存、最高等级、最高击杀、最高伤害、完美胜利、最高威胁通关和最近构筑。
- 威胁等级分为标准、进阶、极限，优先改变波次数量、刷新节奏和特殊怪上限，不额外堆 HP。
- 固定种子挑战使用 `20260710`；通关逐步解锁脉冲、寒霜和压缩三种侧向初始倾向，不做角色系统。
- 已接入四组关键 SFX、一首循环 BGM、声音/震屏/减少动态效果设置，以及特殊怪独立程序化轮廓。
- 已支持横屏单摇杆、Touch 选卡、safe area 和切后台暂停；`build-config/android.json` 已启用左右横屏并关闭竖屏，本轮未产出 APK、未做真机验收。

优先打磨：

- 光环范围、伤害、拾取、移速等基础成长手感。
- 脉冲光环和寒霜光环的节奏、视觉反馈和数值。
- 光环效果必须有可读的视觉区分：脉冲使用金色蓄力环和爆发射线，寒霜使用冰蓝内环和放射冰刺。
- 基础怪、快怪、厚怪、自爆怪这 4 类压力模型。
- 阶段切换时地图色调、怪物池和压力节奏同步变化。

暂时不做：

- 多角色。
- 装备系统。
- 天赋树。
- 排行榜。
- 复杂金币商店。
- 多套完整 Boss 机制。

产品判断：先做一个 8 分钟可重复游玩的 vertical slice。如果这个版本不好玩，继续堆技能、Boss 和局外成长也只是给问题盖被子。

阶段 A～D 的游戏内功能与 Android 横屏构建配置已按 `docs/player-experience-roadmap.md` 落地。当前剩余工作是 APK 产出、真机验收，以及独立的 Web 预览工具链修复。

## 刷怪规则

- 刷怪点围绕玩家生成。
- 怪物生成在屏幕外，距离玩家约 520 到 760 像素。
- 0 到 90 秒：第一关 幽暗草场，影虱为主，少量爆裂孢子。
- 90 到 210 秒：第二关 疾风荒径，加入疾行妖、风剪灵和少量爆裂孢子。
- 210 到 330 秒：第三关 石壳废墟，加入石壳兽并提高障碍密度。
- 330 到 480 秒：第四关 裂隙深处，加入裂隙眼和更多爆裂孢子。
- 90、210、330 秒触发随机波次和精英节点；精英击杀返还一次重投。
- 裂隙核心在 420 秒出现；撑到 480 秒普通胜利，提前击破完美胜利。

## 升级与技能

升级时暂停游戏，随机出现三个技能卡片，玩家选择一个后继续游戏。

优先实现的强化：

- 光环伤害增加。
- 光环半径增加。
- 移动速度增加。
- 最大生命增加。
- 拾取范围增加。
- 脉冲、寒霜和压缩三种行为流派。
- 每局一次重投，阶段精英可返还重投。

后续再加：

- 暴击。
- 中毒。
- 冰冻。
- 灼烧。
- 护盾。
- 吸血。
- 复活。

## 开发原则

- 先做核心闭环，再扩内容。
- 先保证手感和成长反馈，再补美术音效。
- 技能数量不要一开始堆满 20 个，先做 6 到 8 个能证明循环好玩的技能。
- 多 Boss、联网系统、金币、装备和天赋排在 MVP 后面；终局 Boss 与本地纪录已落地。
