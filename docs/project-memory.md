# 《光环生存》项目记忆

## 项目定位

- 游戏名：《光环生存》（暂定），英文名：Aura Survivor。
- 类型：2D Roguelike / 割草生存 / 单机休闲 / 自动攻击。
- 引擎：Cocos Creator 3.8 LTS。
- 语言：TypeScript。
- 首发平台：Windows。
- 后续平台：Android。
- 开发规模：1 人，预计 2 到 3 个月业余开发。

- `docs/project-proposal.md`：游戏立项书。
- `docs/gdd-v1.md`：Game Design Document（GDD）V1.0。

## 核心玩法

玩家控制角色移动，角色周围持续存在光环。怪物不断刷新并追击玩家，进入光环范围后持续受到伤害。怪物死亡掉落经验球，玩家拾取经验后升级，并从三个随机强化中选择一个继续成长。

核心循环：

```text
移动 -> 刷怪 -> 追击 -> 光环伤害 -> 怪物死亡 -> 经验掉落 -> 拾取升级 -> 技能三选一 -> 更强怪物 -> 继续生存
```

游戏目标：尽可能生存更久。

## MVP 范围

V1.0 最小可玩版本只保留这些内容：

- 玩家 WASD / 方向键移动。
- 一个持续伤害光环。
- 普通怪自动追击。
- 怪物死亡和经验掉落。
- 经验升级和三选一强化。
- 生存计时与游戏结束。
- 一张无限循环地图，并按当前关卡名切换主题地表、障碍和色调。

暂不优先做：

- 多角色。
- 装备系统。
- 天赋树。
- 成就系统。
- 独立多地图加载和复杂地图事件。
- 排行榜。

## 产品方向记录

产品定位不做“低配吸血鬼幸存者”，而是做“走位驱动的光环构筑游戏”。核心卖点不是自动攻击本身，而是玩家通过移动和站位，把怪物拖进自己的危险光环里，同时避免被包围。

一句话定位：

```text
《光环生存》是一款无武器、靠站位和光环构筑割草的短局 Roguelite 生存游戏。
```

当前最重要的验证目标：

- 开局 30 秒内让玩家理解玩法。
- 3 分钟内让玩家获得明显成长反馈。
- 5 分钟后让局势开始紧张。
- 8 分钟形成一次完整高潮和胜利结算。

当前 vertical slice 决策：

- 局长目标：撑过 8 分钟即胜利。
- 差异化机制：光环效果优先作为局内升级，不先绑定不同角色。
- 当前光环效果升级：脉冲光环负责周期爆发，寒霜光环负责范围减速。
- 压力模型：基础怪、快怪、厚怪、自爆怪先跑通，裂隙眼作为终局精英压力。
- 后续角色只做初始倾向，例如初始范围更大、脉冲冷却更短或寒霜减速更强，不在 MVP 做角色系统。

优先打磨：

- 光环范围、伤害、拾取、移速等基础成长手感。
- 脉冲光环和寒霜光环的节奏、视觉反馈和数值。
- 光环效果必须有可读的视觉区分：脉冲使用金色蓄力环和爆发射线，寒霜使用冰蓝内环和放射冰刺。
- 基础怪、快怪、厚怪、自爆怪这 4 类压力模型。
- 阶段切换时地图色调、怪物池和压力节奏同步变化。

暂时不做：

- 多角色。
- 装备系统。
- 天赋树。
- 排行榜。
- 复杂金币商店。
- 多套完整 Boss 机制。

产品判断：先做一个 8 分钟可重复游玩的 vertical slice。如果这个版本不好玩，继续堆技能、Boss 和局外成长也只是给问题盖被子。

## 刷怪规则

- 刷怪点围绕玩家生成。
- 怪物生成在屏幕外，距离玩家约 800 到 1200 像素。
- 0 到 55 秒：第一关 幽暗草场，影虱为主，少量爆裂孢子。
- 55 到 115 秒：第二关 疾风荒径，加入疾行妖和风剪灵。
- 115 到 180 秒：第三关 石壳废墟，加入石壳兵和腐泥怪。
- 180 秒后：第四关 裂隙深处，加入裂隙眼和更多爆裂孢子。
- Boss 暂定在 5、10、15 分钟出现。

## 升级与技能

升级时暂停游戏，随机出现三个技能卡片，玩家选择一个后继续游戏。

优先实现的强化：

- 光环伤害增加。
- 光环半径增加。
- 移动速度增加。
- 最大生命增加。
- 拾取范围增加。
- 经验获取增加。

后续再加：

- 暴击。
- 中毒。
- 冰冻。
- 灼烧。
- 护盾。
- 吸血。
- 复活。

## 技术模块

计划模块：

- Player：玩家移动、属性、受伤、死亡。
- Enemy：怪物属性、追击、受伤、死亡。
- Aura：光环范围、Tick 检测、持续伤害。
- Spawner：刷怪时间轴、刷怪位置、怪物权重。
- Experience：经验球生成、吸附、拾取。
- Upgrade：升级暂停、三选一界面、选择结果。
- Skill：技能数据和效果应用。
- GameManager：游戏状态、计时、击杀数、结算。
- UI：HUD、暂停、升级、结算。
- Save：最高分、最高等级、总金币、设置。

## 开发原则

- 先做核心闭环，再扩内容。
- 先保证手感和成长反馈，再补美术音效。
- 技能数量不要一开始堆满 20 个，先做 6 到 8 个能证明循环好玩的技能。
- Boss、存档、金币、装备、天赋都排在 MVP 后面。
