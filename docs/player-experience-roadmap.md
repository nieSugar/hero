# 《光环生存》重度玩家体验审查与迭代路线

- 状态：阶段 A～D 与原生 Android 横屏构建配置已完成；APK 产出、真机验收和 Web 预览工具链仍待单独处理
- 记录日期：2026-07-10
- 适用版本：Cocos Creator MVP `0.1.0`
- 核心目标：先把 8 分钟局做成公平、可读、能形成不同构筑的完整闭环，再扩局外系统和内容。

## 1. 审查范围与验证边界

本次审查覆盖：

- `assets/scripts/AuraSurvivor*.ts` 的入口、Session、输入、刷怪、战斗、经验、升级、Boss、UI、存档与绘制逻辑。
- `assets/Game.scene` 的场景和节点结构。
- 两张 Sprite 资源及其导入配置。
- `docs/gdd-v1.md`、`docs/project-proposal.md`、`docs/project-memory.md` 的产品目标与当前实现差异。
- Cocos 本地预览服务、场景资源请求和 TypeScript 静态检查。

验证边界：职责重构前，Cocos 原生 Simulator 已成功加载 `Game` 场景并完成状态级冒烟与窗口截图验收；重构后已通过 TypeScript 编译、独立 Session 自检和组合生命周期冒烟，Creator Simulator 尚待回归。Touch、safe area、切后台暂停和横屏 HUD 已在游戏内实现，`build-config/android.json` 已启用左右横屏并关闭竖屏。本轮未产出 APK、未做真机验收。Web 预览仍受本机 Cocos 3.8.8 custom pipeline Effect 数据异常影响，属于独立工具链问题，不作为玩法代码失败处理。

## 2. 总结论

当前版本已完成从公平性、构筑、波次复玩到基础表现和 Touch 输入的 vertical slice；本节至第 6 节保留为实施前审查快照，当前结果以第 7 节复选框和第 12 节实施记录为准。

以下 `AuraSurvivorGame.ts:<旧行号>` 均指 2026-07-10 职责拆分前版本，仅作为历史审查证据，不用于定位当前源码；当前实现请按 `AuraSurvivorModel`、`AuraSurvivorSession` 与 `AuraSurvivorRenderer` 中的配置和方法名检索。

真正短板不是内容数量，而是四个闭环尚未成立：

1. **站位闭环**：均匀圆形光环让最优操作容易收敛成持续绕圈，光环内具体站位缺乏技术差异。
2. **构筑闭环**：9 项强化中只有脉冲、寒霜、压缩明显改变行为，其余主要是裸数值；重复选择后构筑容易趋同。
3. **压力闭环**：刷怪频率、批次数量和属性成长同时叠加，后期容易从“策略压力”变成“不可读怪海”。
4. **奖励闭环**：Boss 可以无视，结算缺少 Boss 状态、死因、完整构筑、最佳纪录和下一局目标。

产品方向继续保持：

> 不做“低配吸血鬼幸存者”，而是做“走位驱动的光环构筑游戏”。

现阶段不应通过增加传统武器系统掩盖光环本身的深度问题。

## 3. 当前值得保留的基础

- 8 分钟短局目标明确，适合作为 vertical slice。
- 光环范围始终可见，玩家能够理解基础攻击区域。
- 自爆孢子和裂隙脉冲拥有提前预警，公平设计方向正确。
- 脉冲与寒霜已经具备不同视觉语言和强化逻辑。
- 敌人、特效已有硬上限；审查时单组件结构适合快速验证，功能完成后已按真实职责拆分。
- 升级支持数字键和鼠标，并能连续处理溢出经验。
- 第 25 秒前不刷自爆怪，避免开局直接阴人。

这些基础应继续复用，不急着重构成 Player、Enemy、Spawner 等一地文件。

## 4. 玩家体验时间线

| 时间 | 预期玩家感受 | 当前风险 |
| --- | --- | --- |
| 0～30 秒 | 理解移动、自动攻击、引怪入环和拾取经验 | 菜单未说明 WASD、自动攻击、拾取和 ESC；两层范围缺少解释 |
| 1～3 分钟 | 获得第一个明显流派能力 | 首次升级可能全是裸数值，流派感出现太晚 |
| 3～4 分钟 | 压力逐渐增加，开始考验控距 | 210 秒附近刷怪量断崖增长，容易突然失控 |
| 5～7 分钟 | 构筑成型，通过特殊怪组合验证强度 | 特殊怪按普通权重生成，预警可能堆成视觉噪音 |
| 7～8 分钟 | Boss 形成终局高潮 | Boss 可以完全不打，击杀也没有专属奖励或结算 |
| 结算 | 理解本局强弱、死亡原因，并产生下一局目标 | 只有等级和击杀，不能返回菜单，也没有纪录或解锁 |

## 5. 问题清单与产品决策

### P0：公开试玩前处理

#### P0-01 终局 Boss 与胜利脱节

证据：

- 480 秒直接进入 `victory`：`AuraSurvivorGame.ts:216-234`。
- Boss 在 420 秒出现：`AuraSurvivorGame.ts:289-319`。
- Boss 死亡只走普通击杀和经验球逻辑：`AuraSurvivorGame.ts:523-529`。
- Boss 使用固定伤害，而 420 秒后的普通裂隙眼和自爆怪继续按时间成长，终局杂兵可能比 Boss 单次攻击更危险。

玩家影响：最优策略可以变成无视 Boss、继续绕圈，终局高潮只是演出。

决策：

- 撑过 8 分钟保留为普通胜利。
- 最后一分钟击杀裂隙核心记为完美胜利，并提供独立纪录或解锁。
- 终局阶段降低普通怪噪音，让 Boss 成为主要威胁。
- Boss 至少增加一个明确阶段变化和专属击杀反馈。
- 终局 Boss 与杂兵使用同一套威胁预算，避免一边强化 Boss、一边让普通怪继续指数堆噪音。

#### P0-02 难度存在断崖和叠怪瞬杀

证据：

- 阶段与刷怪配置：`AuraSurvivorGame.ts:118-123`。
- 刷怪量同时叠加阶段批次、`floor(aliveTime / 110)` 和间隔缩短：`AuraSurvivorGame.ts:347-355`。
- 约 209 秒为 `5.4` 只/秒，210 秒跳到 `13.2` 只/秒；330 秒约 `31.25` 只/秒，末期理论请求约 `37.5` 只/秒。
- 每只怪拥有独立的 `0.6s` 接触伤害冷却，怪物之间没有分离：`AuraSurvivorGame.ts:389-430`。
- 敌人更新先于本帧光环伤害，贴身敌人可能先完成接触伤害，再被光环击杀。

玩家影响：死亡可能来自同帧叠加伤害，而不是可理解、可复盘的操作失误。

决策：

- 使用一条连续压力曲线，阶段切换只负责怪物池、主题和波次变化。
- 为裂隙眼、自爆怪设置独立生成冷却和同时在场上限。
- 玩家受伤后增加短暂全局保护，并提供闪白、击退和音效反馈。

#### P0-03 构筑存在支配项和陷阱项

证据：

- 普通伤害牌为 `×1.25`，压缩光环为 `×1.45` 且半径最低锁在 68：`AuraSurvivorGame.ts:612-642`。
- 半径到下限后继续选择压缩光环不再支付代价。
- 经验牌消耗一次升级，只增加后续 15% 经验，而升级需求每级约增长 45%：`AuraSurvivorGame.ts:596-620`。
- 指定强化每次有约三分之一概率出现，完整局内构筑容易收敛到脉冲加寒霜。

决策：

- 压缩光环改成一次性流派选择，选后移出卡池。
- 首次升级保证出现至少一个行为牌。
- 强化增加最大等级，并根据已选流派过滤后续卡池。
- 经验牌先移出 8 分钟卡池，或改成“立即经验 + 小幅长期增益”。
- “霜爆光环”必须获得真实联动，例如寒霜目标被脉冲命中后碎冰爆炸。

#### P0-04 信息层级与性能风险

证据：

- 每只怪固定掉一个经验球，Orb 无上限、合并或过期：`AuraSurvivorGame.ts:523-529`、`579-594`。
- 每只怪常驻血条，敌人上限为 180：`AuraSurvivorGame.ts:720-745`。
- 影虱、疾行妖、裂隙眼、自爆孢子和 Boss 大量共用 `enemy_eye`：`AuraSurvivorGame.ts:1192-1194`。
- `Graphics` 位于父 Canvas，运行时 Sprite 和 Label 是其子节点：`Game.scene:400-438`、`AuraSurvivorGame.ts:1168-1220`。
- `deltaTime` 被截断到 `0.05s`，生存计时也使用截断值；低于 20 FPS 时，真实局长会超过 8 分钟：`AuraSurvivorGame.ts:216-234`。
- 两张像素资源原先使用 `linear` 过滤和 `repeat`，缩放后容易发糊并产生 atlas 边缘采样风险。

玩家影响：后期奖励、血条、预警与同轮廓怪物混在一起；暂停、升级和结算遮罩也可能压不住实体 Sprite。

决策：

- 邻近经验球自动合并，限制实体数量，并增加屏外绘制裁剪。
- 普通怪仅在受伤后短暂显示血条，精英和 Boss 常驻。
- 优先给自爆怪、裂隙眼和 Boss 独立轮廓。
- 固定渲染层：背景 → 实体 → 前景/特效 → HUD → Modal Overlay。
- 将真实局时钟与受控模拟步长分开，低帧率不能改变胜利时长和难度节奏。
- 像素资源改为 `nearest`，atlas 改为 `clamp`。

#### P0-05 基础流程不闭合

证据：

- 菜单只说明目标和开始方式：`AuraSurvivorGame.ts:929-936`。
- 暂停只有“按 ESC 继续”：`AuraSurvivorGame.ts:684`。
- 游戏结束和胜利只有重开：`AuraSurvivorGame.ts:951-960`。

决策：

- 菜单增加移动、自动攻击和暂停说明。
- 首局依次提示移动、引怪入环、拾取经验，完成后立即消失。
- 共用暂停面板提供继续、重新开始、返回菜单。
- 结算增加完整强化层数、Boss 状态、总伤害、死因和个人最佳。
- 应用失焦时自动暂停并清空按键集合，避免切回窗口后角色继续移动。

### P1：完成可重复游玩的 vertical slice

#### P1-01 光环需要真实站位深度

当前均匀圆形光环只要求敌人在范围内，重度玩家容易将操作解成持续绕圈。

优先验证的最小机制：

- 外缘作为高伤或充能甜点区，内圈偏控制。
- 敌人跨越边界时为脉冲充能或增加连击收益。
- 寒霜与脉冲形成碎冰联动。
- 压缩流派用更小安全距离换取爆发伤害。

这些机制复用现有距离计算，不增加传统主动攻击系统。

#### P1-02 用现有敌人组成波次

当前刷怪只是从固定池逐只随机，第二局以后时间轴高度可预测。

先复用现有 5 种怪组成：

- 快怪突袭。
- 石壳护送。
- 自爆包围。
- 裂隙眼炮台波。

每阶段随机选择少量模板，并在 90、210、330 秒设置一个明确事件或精英奖励节点。

#### P1-03 阶段不能只换色

四阶段共用障碍算法和碰撞规则，主要差异是色调和怪物池。

不制作四张独立地图，先让现有生成器按主题改变：

- 障碍密度和尺寸比例。
- 无碰撞装饰。
- 一种轻量阶段规则，例如风径开阔、废墟窄道、裂隙危险地脉。

#### P1-04 高风险敌人奖励与有效 TTK 不匹配

330 秒附近，疾行妖约为 36 HP、17 经验，裂隙眼约为 566 HP、55 经验；裂隙眼的经验/HP 收益约只有疾行妖的五分之一，却拥有范围攻击。

石壳兽的减伤持续 `0.42s`，光环每 `0.3s` 命中一次且每次重新设置减伤，因此持续处于光环内时几乎永久保持减伤：`AuraSurvivorGame.ts:485-520`。

决策：

- 石壳防御改成带独立冷却的短窗口，减伤中的命中不能续期。
- 精英奖励按有效 TTK、攻击威胁和特殊能力重标。
- 高风险怪优先掉多颗合并经验、强化奖励或重投资源，避免最优策略变成只刷快怪、拖着厚怪不管。

#### P1-05 文本与构筑信息需要可读

所有 Label 使用约 `字号 × 1.4` 的单行高度，同时启用换行和 `SHRINK`；长升级描述可能被压得过小：`AuraSurvivorGame.ts:1196-1212`。

决策：

- 升级描述提供固定的 2～3 行高度。
- 底部强化历史按名称聚合层数，不再只展示最近四次重复文本。
- 预警除颜色外增加轮廓、虚线或图标，不能要求玩家只靠色相区分。

### P2：核心循环稳定后处理

- 本地最佳时间、最高等级、最高击杀和 Boss 击杀纪录。
- 威胁等级和固定种子挑战。
- 3 个侧向初始光环倾向，不做直接数值碾压。
- 受伤、击杀、拾取/升级、Boss 预警四组基础 SFX 与一首循环 BGM。
- 关闭震屏、减少动态效果、形状加颜色的预警表现。
- Android 强制横屏、单摇杆、Touch 选卡、safe area 和切后台暂停。
- 自爆怪、裂隙眼、Boss 的独立美术，再扩风剪灵和腐泥怪。

当前已支持横屏单摇杆、Touch 选卡/按钮、safe area 和切后台暂停；Android 构建配置已锁定左右横屏，发布前仍需产出 APK 并完成真机验收。

## 7. 分阶段实施计划

原则：一次只实现一个可独立验证的小任务；每项完成后保持游戏可运行，再进入下一项。

### 阶段 A：公平性与终局闭环

目标：玩家能够清楚理解为什么受伤、为什么死亡，以及 Boss 是否被击败。

- [x] A01 固定 Background、Entity、FX、HUD、Overlay 渲染层级。
- [x] A02 增加玩家全局受击保护和受击反馈。
- [x] A03 将刷怪强度改为连续曲线。
- [x] A04 增加裂隙眼和自爆怪的生成冷却与在场上限。
- [x] A05 增加普通胜利与 Boss 完美胜利状态。
- [x] A06 为 Boss 增加阶段变化、专属死亡反馈和结算字段。
- [x] A07 合并邻近经验球，并限制 Orb 实体数量。
- [x] A08 完成暂停、返回菜单和结算复盘闭环。
- [x] A09 分离真实局时钟与受控模拟步长。

验收：

- 不会因多只怪同帧接触而无反馈瞬死。
- 210 秒和 330 秒阶段切换不再造成刷怪量断崖。
- 特殊怪预警数量始终可读。
- 结算明确显示普通胜利、完美胜利或失败原因。
- 长时间不拾取经验时 Orb 实体数量不会无限增长。
- 在低于 20 FPS 的压力测试中，8 分钟目标仍按真实时间推进。

### 阶段 B：三条光环构筑

目标：玩家在 90 秒内形成流派方向，三局之间有明显不同的操作重点。

- [x] B01 首次升级保证出现脉冲、寒霜、压缩之一。
- [x] B02 压缩改为一次性流派选择并修正重复收益。
- [x] B03 给所有强化增加等级、上限和动态描述。
- [x] B04 根据当前流派过滤升级池。
- [x] B05 实现寒霜加脉冲的碎冰联动。
- [x] B06 实现光环甜点区与跨边界收益。
- [x] B07 增加每局一次重投；阶段精英奖励来源随阶段 C 的事件系统接入。
- [x] B08 修正石壳减伤窗口和高风险敌人奖励。

验收：

- 第一次升级必有行为变化选项。
- 不存在半径到下限后继续无代价指数增伤。
- 脉冲、寒霜、压缩分别形成不同站位策略。
- 卡片展示当前等级、变化前后数值和是否已达上限。
- 高风险敌人的奖励与其有效 TTK、特殊能力相匹配。

### 阶段 C：局内节奏与复玩

目标：第二、第三局的敌群节奏和中途目标不完全相同。

- [x] C01 为现有敌人建立少量波次模板。
- [x] C02 在 90、210、330 秒加入阶段事件或精英节点。
- [x] C03 精英奖励行为牌、进化或重投资源，而不只是经验。
- [x] C04 让四阶段拥有不同障碍密度、装饰和轻量规则。
- [x] C05 增加本地最佳纪录与完整构筑记录。
- [x] C06 增加威胁等级，优先修改敌人组合与节奏，不只堆 HP。

验收：

- 连续三局至少出现不同波次组合。
- 每 1～2 分钟存在一个清晰决策或奖励节点。
- 玩家能够在结算中比较本局与个人最佳。

### 阶段 D：表现、内容与平台

目标：在核心循环已证明可重复游玩后，提高完成度并准备目标平台。

- [x] D01 加入四组关键 SFX 和一首循环 BGM。
- [x] D02 增加震屏开关和减少动态效果选项。
- [x] D03 将像素资源过滤改为 `nearest`，atlas 改为 `clamp`。
